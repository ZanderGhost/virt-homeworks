Посмотрите на сценарий ниже и ответьте на вопрос: "Подходит ли в этом сценарии использование докера? Или лучше подойдет виртуальная машина, физическая машина? Или возможны разные варианты?"

Детально опишите и обоснуйте свой выбор.

Сценарий:

- Высоконагруженное монолитное java веб-приложение;

        Лучше выбрать виртуальную машину, так как в  VM есть возможность динамического  
        выделения ресурсов, есть возможность добавить память или увеличить количество ядер 
        процессора, можно развернуть несколько инстансов. Так же с VM можно сделать слепок системы для востановления в 
        случае возникщих проблем.
- Go-микросервис для генерации отчетов;

        В данном случае контейтер Docker лучший выбор - малый вес, запустился, 
        выполнил свои дела и умер.
- Nodejs веб-приложение;

       Здесь подойдет docker
- Мобильное приложение c версиями для Android и iOS;

        В принципе имет место быть 

- База данных postgresql используемая, как кэш;

        В данном случае лучше выбрать железный сервер, так как потребуется 
        быструй отклик от сервера.
- Шина данных на базе Apache Kafka;

        В данном случае в чистом виде docker мало подходит, так как мы имеем 
        большое колличество данных, что сводит на нет приимущества применения
        контейнеров. Так же при перезапуске контейнеров потеряются данные. 
        Еще онда причина не использовать docker это значительное количество 
        оперативной памяти потребляемое Kafka. В данном случае подойдет кластер из VM.
- Очередь для Logstash на базе Redis;

        Для очереди docker вполне логично использовать.
- Elastic stack для реализации логирования продуктивного веб-приложения - три ноды elasticsearch, два logstash и две ноды kibana;
        
        Docker  отлично подходит для ELK  стека учитывая репликацию данных между нодами  ELK. 
        
- Мониторинг-стек на базе prometheus и grafana;
        
        Для сбора метрик и их вызуализации docker отлично подходит т.к. 
        тут важнее возможность горизонтального маштабирования, чем сохранность данных.
- Mongodb, как основное хранилище данных для java-приложения;
        
        Как хранилище данных контейнеры нет смысла использовать, так как при их
        перезапусте данные будут утеряны. Лучше подохдет кластер из нескольких VM.
       
- Jenkins-сервер.
    
        Docker отлично подходит - запустился при деплое, отработал и дальше спать.
